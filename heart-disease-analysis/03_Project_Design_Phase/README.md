# 🏗️ 03 – Project Design Phase

## 📖 Overview

The Project Design Phase transforms the project requirements into a structured solution. During this phase, the overall system design, solution approach, and architecture are planned before implementation begins.

The design focuses on developing an efficient workflow for analyzing heart disease data using SQL, creating interactive visualizations with Tableau, and integrating the dashboard into a Flask web application. A well-planned design ensures that the project is scalable, organized, and capable of delivering meaningful healthcare insights.

---

# 🎯 Objectives

The objectives of this phase are:

- Design an effective solution for heart disease data analysis.
- Define how the proposed solution addresses the identified problem.
- Plan the overall architecture of the system.
- Organize the interaction between different project components.
- Prepare a blueprint for the development phase.

### Description

This document explains how the proposed solution directly addresses the identified healthcare problem.

The project analyzes heart disease data using SQL queries to identify important trends and risk factors. The analyzed data is then visualized through Tableau dashboards and presented to users using a Flask-based web application.

---

## 📄 Proposed Solution

**File:** `Proposed_Solution.pdf`

### Description

The proposed solution describes the complete approach followed to build the Heart Disease Analysis Dashboard.

The workflow consists of:

- Collecting the heart disease dataset.
- Importing the dataset into MySQL.
- Performing SQL queries to analyze healthcare data.
- Creating interactive dashboards in Tableau.
- Embedding the Tableau dashboard into a Flask web application.
- Providing users with an interactive platform for exploring healthcare insights.

This approach combines database management, data analysis, visualization, and web deployment into one integrated solution.

# 🏛️ Design Highlights

The project design follows a modular approach where each component performs a specific task.

### Data Layer

- Stores the healthcare dataset in MySQL.
- Supports SQL-based analysis.

### Analysis Layer

- Executes SQL queries.
- Generates summarized healthcare information.

### Visualization Layer

- Builds interactive dashboards using Tableau.
- Presents charts, KPIs, and healthcare insights.

### Presentation Layer

- Displays the Tableau dashboard through a Flask web interface.
- Provides users with an easy-to-use dashboard experience.
