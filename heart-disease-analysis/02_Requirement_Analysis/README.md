# 📋 02 – Requirement Analysis

## 📖 Overview

The Requirement Analysis phase is one of the most important stages of the Heart Disease Analysis project. During this phase, the project requirements are carefully gathered, analyzed, and documented before moving to the design and development stages.

The purpose of this phase is to understand the expectations of healthcare professionals, researchers, and end users while defining the complete workflow of the system. Proper requirement analysis minimizes development risks and ensures that the final dashboard effectively addresses the project objectives.

This phase also identifies the technologies, system architecture requirements, and the complete data flow used throughout the project.

---

# 🎯 Objectives

The major objectives of this phase are:

- Understand the business problem and user requirements.
- Identify the functional and non-functional requirements.
- Design the complete data flow of the system.
- Select suitable technologies for implementation.
- Define how users will interact with the dashboard.
- Prepare documentation for the design and development phases.

- Discovering the dashboard

---

## 📄 2. Solution Requirement

**File:** `Solution_Requirement.pdf`

### Description

This document defines all the requirements needed for successful implementation of the project.

### Functional Requirements

The system should:

- Import healthcare dataset
- Store data in MySQL database
- Execute SQL queries
- Generate interactive Tableau dashboards
- Embed Tableau dashboard into Flask application
- Display KPIs and visualizations
- Allow users to interact with filters
- Present healthcare insights

### Non-Functional Requirements

The system should provide:

- Fast dashboard loading
- High usability
- Responsive interface
- Data accuracy
- Security
- Reliability
- Easy maintenance
