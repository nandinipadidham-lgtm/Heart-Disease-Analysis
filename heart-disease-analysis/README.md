# Heart Disease Analysis

A data analytics project analyzing heart disease data, following a structured project lifecycle from ideation through final demo.

## Repository Structure

| Folder | Phase | Contents |
|---|---|---|
| `01_Ideation_Phase` | Ideation | Problem statement, empathy map, brainstorming |
| `02_Requirement_Analysis` | Requirement Analysis | Customer journey map, solution requirement, data flow diagram, tech stack |
| `03_Project_Design_Phase` | Project Design | Problem-solution fit, proposed solution, solution architecture |
| `04_Project_Planning_Phase` | Project Planning | Project planning template |
| `05_Project_Development_Phase` | Project Development | Preprocessing steps, business questions with visualization, dashboard screenshots, dataset |
| `06_Performance_Testing` | Performance Testing | Performance testing template/results |
| `07_Doc_and_Demo` | Documentation & Demo | Final report, demonstration video |

## Project Overview

- **Project name:** Heart Disease Analysis

- **Status:** In progress

## How to Use This Repo

Each folder contains a `README.md` describing exactly what belongs there, plus placeholder files where relevant. Fill each phase in as you complete it, then commit and push.

```bash
git add .
git commit -m "Update <phase name>"
git push
```
